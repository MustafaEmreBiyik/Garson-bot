# Processed Datasets

This directory contains JSONL files generated by `training/dataset_builder.py`.

## Files

| File | Description |
|---|---|
| `waiter_sft_train.jsonl` | Training split (~85% of seed examples) |
| `waiter_sft_valid.jsonl` | Validation split (~15% of seed examples) |

## JSONL Format

Each line is a JSON object in chat-training format:

```json
{
  "messages": [
    {"role": "system", "content": "...system prompt..."},
    {"role": "user", "content": "...customer utterance..."},
    {"role": "assistant", "content": "...ideal waiter response..."}
  ],
  "metadata": {
    "id": "add_001",
    "intent": "add_item",
    "entities": {"item": "Ayran", "quantity": 2}
  }
}
```

## Regenerating

```powershell
.venv\Scripts\python.exe -m robot_waiter_ai.training.dataset_builder
```

The source of truth is always `datasets/raw/seed_dialogues.yaml`. These processed files
should be regenerated from the raw dataset whenever text cleanup or example corrections
are made.
