# Development Rules

1. Stay within the conversational AI MVP scope.
2. Do not add navigation, SLAM, ROS, or robot control features.
3. Keep modules small, deterministic, and testable.
4. Keep structured restaurant knowledge in YAML rather than hard-coding it in prompts.
5. Preserve the polite, concise waiter persona in Turkish.
6. Add tests for core behavior changes whenever practical.
7. Default language is Turkish unless a future configuration explicitly changes it.
8. Do not start model training or external model integration unless explicitly requested.
